## 言語方針

- プロジェクト内のドキュメント、UI 文言、コメント、コミット説明、Issue / PR の本文は、可能な限り日本語を優先して使用してください。
- 技術用語、ライブラリ名、API 名、コード識別子、設定キーなど、日本語化すると意味が不明確になるものは英語のままで構いません。
- 既存の英語テンプレートや外部仕様に従う必要がある箇所は、英語を使用して構いません。
- 新しく追加するユーザー向け文言は、原則として日本語で記述してください。


## コミットメッセージ

- コミットメッセージは例外として English semantic style で記述してください。
- 形式は `type(scope): summary` を基本とします。例: `feat(frontend): add Next.js 15 app shell`
- `type` は `feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore` などを使用してください。
- summary は命令形・現在形の短い英語で記述してください。

## 開発方針

- Next.js は 15 系に固定してください。
- 秘密情報は `.env.local` などの環境変数ファイルに置き、リポジトリへコミットしないでください。
- 変更後は、可能な範囲で `npm run lint`、`npm run typecheck`、`npm run build` を実行して確認してください。
