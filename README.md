## プロダクト概要

> Day1 のテーマ確定後に記入。3-5 行でプロダクトを説明。

食事制限（アレルギー、グルテンフリー、ヴィーガンなど）を持つ人と、海外料理愛好家の偏愛を掛け合わせ、日本の食材で世界の味を再現するプラットフォームです。制限のために食の選択肢を狭めている人には「安全に食べられる未知の海外料理」を提案し、現地の味が恋しい愛好家には「日本のスーパーの食材で本場を再現するレシピ」を提供します。将来的には、お互いの調理の知恵や経験談を共有し合う自立型の相互扶助コミュニティを目指します。

### 解決したい課題
- 食事制限を持つ人： 自炊に頼らざるを得ず食がマンネリ化している。馴染みのない海外料理が安全に食べられるかの検索・調査（原材料のトレース）に膨大な時間と労力がかかる。制限は人それぞれ多種多様なため、身近に同じ状況の仲間がおらず情報が不足している。
- 海外料理愛好家： 過去に海外で慣れ親しんだ大好きなローカルフードを日本でも食べたいが、専門のお店がなく、自分で作ろうにも本場の食材が国内で手に入らないため再現方法がわからない。

### ターゲットユーザー
- 食事制限ユーザー（グルテン、アレルギー等）： 外食は原材料が不透明で友達に気を遣うため自炊がメインだが、レパートリーが狭く食の楽しさを諦めかけている。自分と同じアレルギーレベルの人がどう料理を工夫しているか、リアルな情報に飢えている。
- 海外料理愛好家（帰国子女・留学生）： 過去に海外に住んでいて、今日本に住んでいる。自分の思い出の味を再現したいが、現地のスパイスや調味料の「日本のスーパーで買える代替品」が見つけられず模索している。


### コア機能
日本の身近な食材で世界の味を再現したレシピを蓄積・検索できるデータベース機能。ユーザーの状況に応じて以下の使い方ができ、将来的にはコメント等で代替案を出し合うコミュニティへ発展させます。
- 食事制限ユーザー： アレルギーなど自分の食事制限内容を登録すると、その制限をクリアした、自分が今まで知らなかった未知の海外料理レシピのエントリーが一覧で並び、食の視野が広がる。
- 海外料理愛好家： 再現したい料理名や国／地域のカテゴリを登録すると、現地の特殊な食材を使わず、日本のスーパーで買える食材に置き換えて完全再現されたレシピが見つかる。
- 重複ユーザー： 海外料理に対して、さらに自分の食事制限をクリアしているレシピエントリーをピンポイントで引き出す。

## セットアップ手順

フロントエンドは Next.js 15 系に固定しています。Supabase は Web 上のクラウドプロジェクトを Supabase CLI でリンクして利用します。

```bash
# 1. クローン
git clone https://github.com/engineer-guild-hackathon-2026-05/team-04.git
cd team-04

# 2. 依存関係をインストール
npm install

# 3. Supabase Web プロジェクトに接続
supabase login
supabase link --project-ref kiicjqiylsmlxupvhrti

# 4. 開発サーバーを起動
npm run dev
```

環境変数は `.env.local` に保存し、リポジトリへコミットしないでください。

```bash
NEXT_PUBLIC_SUPABASE_URL=https://<project-ref>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-key>
```

品質確認には以下を使用します。

```bash
npm run lint
npm run typecheck
npm run build
```
